# #007 — `LocalStyle` `##` 정규식 에지 케이스

- **심각도**: 🟢 에지케이스
- **파일**: `estreU0EEOZ.js` — `LocalStyle.localize()` (L941)
- **해결 버전**: —

## 현상

현재 `##` 치환 정규식:

```js
// 주석 처리된 이전 버전:
// const localizedStyles = styles.replace(/^([\t\s]*)##/gm, "$1" + localPrefix);

// 현재 버전:
const localizedStyles = styles
    .replace(/([^#])##([^#])/g, "$1" + localPrefix + "$2")
    .replace(/^##([^#])/gm, localPrefix + "$1");
```

두 단계로 동작:
1. `([^#])##([^#])` — 앞뒤가 `#` 이 아닌 `##` 치환 (행 중간).
2. `^##([^#])` — 행 시작의 `##` 치환.

## 에지 케이스

### 1. `##` 뒤에 아무것도 없는 경우
```css
## { color: red; }   /* ✅ 동작 — ##뒤에 공백 */
##{ color: red; }    /* ✅ 동작 — ##뒤에 { */
##                   /* ❌ 미치환 — ##뒤에 줄바꿈만 */
```
행 끝에 `##` 만 있으면 `([^#])` 가 매칭할 문자가 없어서 미치환. 실사용에서는 `##` 뒤에 반드시 셀렉터나 `{` 가 오므로 사실상 발생하지 않음.

### 2. `###` (CSS 에서 쓸 일 없음)
`###` 은 양쪽 정규식 모두 `[^#]` 조건에 걸려 미치환. CSS 에서 `###` 은 유효한 표현이 아니므로 문제없음.

### 3. 연속 `##` 두 번 (같은 행)
```css
## > .a, ## > .b { color: red; }
```
첫 번째 replace 에서 `## > .a, ` 의 `##` 가 치환되지만, 두 번째 `##` 앞의 문자가 `,` 로 `[^#]` 에 매칭되므로 정상 치환. **문제없음**.

## 결론

이론적 에지 케이스는 있으나, CSS 작성 관행상 `##` 뒤에 반드시 공백이나 셀렉터가 오므로 **실제 영향은 없다**. 다만 코드에 주석으로 이 동작을 명시하면 유지보수에 도움이 될 수 있음.

## 제안 (선택적)

더 단순한 정규식으로 교체 가능:

```js
const localizedStyles = styles.replace(/(^|[^#])##(?!#)/gm, "$1" + localPrefix);
```

lookahead `(?!#)` 를 사용하면 한 번의 replace 로 처리 가능.
